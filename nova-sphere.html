What It Actually Does.

Your AI agent gets a physical output device. Here's what that looks like in practice.

🤖

Claude Code shows its work

Your agent spawns Claude Code on a GitHub issue. Via MCP, it pushes a progress widget — "Step 3/7, scanning stale recovery..." The face shows busy. When the PR is ready: a chime, notification, and the face switches to excited.

📊

IoT dashboards update in real time

A 20-line bash script pulls data from Prometheus and pushes 4 gauge arcs — temp, pH, salinity, ORP. A Python script connects via MQTT to your Bambu X1C and shows nozzle temp, bed temp, print progress. Swipe between pages.

🗣️

Tap to talk — voice runs locally

Tap the sphere. INMP441 mic captures audio → WebSocket stream to companion server → faster-whisper (STT) → Ollama/gemma3 (LLM) → edge-tts (TTS) → speaker. Swap any component. No cloud dependency unless you want one.

☀️

OpenClaw morning briefing

Your OpenClaw agent runs a heartbeat check: weather, unread emails, today's tasks, reef sensors. It pushes each to the display, narrates via TTS, and ends with "Hey Nova, ask me anything." All orchestrated by the agent, no human input.

REST API

# Set emotion + show notification

curl http://nova.local/face \

-d '{"emotion":"thinking",

"notification":"Building..."}'


# Push a widget dashboard

curl http://nova.local/widgets?page=0 \

-d '{"widgets":[

{"id":"t","type":"arc","value":75,

"indicator_style":{"arc_color":"#00FF88"}},

{"id":"v","type":"label","text":"75%",

"style":{"text_font":"montserrat_48"}}]}'


# Play audio

curl http://nova.local/speaker/play?vol=80 \

--data-binary @alert.wav

Voice Pipeline

Tap → INMP441 (16kHz I2S)

→ faster-whisper (STT, local)

→ Ollama/gemma3 (LLM, local)

→ edge-tts (48kHz neural)

→ Speaker (full duplex)

Each component is swappable. Use Piper for fully offline TTS, any OpenAI-compatible API for LLM, etc.
