Your Agent Decides What It Does.

Nova Sphere is an interface, not an app. Your AI agent decides what to show, say, and do. Here's what the building blocks look like.

Show anything on the display

Widgets, dashboards, charts, text, progress bars — your agent creates them on the fly via API. Up to 5 swipeable pages.

Express emotions

20 animated face states — happy, thinking, angry, sleepy, and more. Your agent sets them to reflect its mood or status.

Listen and speak

Built-in mic and speaker. Tap to talk, your agent responds with voice. STT, LLM, and TTS all run locally — swap any component.

Push notifications

Your agent sends alerts, status updates, reminders — anything. A chime plays, the face reacts, and a message appears on screen.

Play sounds

Tones, alerts, WAV files, TTS audio — your agent plays whatever it needs through the built-in speaker.

Sense touch and motion

Capacitive touch screen and 6-axis IMU. Tap to trigger actions, tilt to navigate — your agent decides what gestures mean.

Wear any face

4 built-in face styles (OpenClaw, Claude, Ollama, default). Add your own in C++ — eyes, mouth, eyebrows, extras are all customizable.

Flash firmware over WiFi

OTA updates via API or MCP. Your agent can even update its own firmware. Full source code included — fork and modify anything.

Control from your phone

Built-in web dashboard at nova.local — set emotions, push widgets, adjust volume, test weather, trigger voice. Works on any phone browser.

No lock-in, no cloud, no subscription

REST API, MCP, WebSocket — pick your protocol. Works with any language, any AI, any automation. You own the hardware and the code.

These are building blocks, not pre-built features. Your AI agent combines them however it wants — that's the point.

Tap → INMP441 (16kHz I2S)

→ faster-whisper (STT, local)

→ Ollama/gemma3 (LLM, local)

→ edge-tts (neural TTS)

→ Speaker (I2S, 48kHz playback)

FAQ

Do I need a server to use it?

For basic features (face emotions, weather, widgets via REST API) — no, just HTTP calls from any device on your network. For voice interaction, you need a companion server running the STT/LLM/TTS pipeline. A Raspberry Pi 4 or any Linux box with a GPU works. The companion server is included in the source code.

Does it work without internet?

Yes — except for weather data and edge-tts (which uses Microsoft's free neural voices). For fully offline operation, use Piper TTS instead. The REST API, face, widgets, and mic all work on local WiFi with zero internet dependency.

What's MCP and why does it matter?

MCP (Model Context Protocol) is an open standard by Anthropic that lets AI agents use tools. Nova Sphere's MCP server exposes 21 tools — so Claude Code, Cursor, Windsurf, or any MCP-compatible agent can natively control the display, play audio, push widgets, and even flash firmware. No integration code needed.

What AI systems does it work with?

Anything that can send HTTP requests. Natively: OpenClaw (built-in emotion sync), Claude Code/Cursor (via MCP), Ollama (voice pipeline). Via REST API: Home Assistant, Node-RED, ChatGPT, your own Python/JS/Go code. If it speaks HTTP, it works.

How big is it?

About 60mm diameter — roughly the size of a tennis ball. Small enough for any desk, large enough for the 1.43" AMOLED to be readable from arm's length.

Is the firmware open source?

Every Nova Sphere comes with the full firmware source code. You can modify, extend, and flash your own builds via OTA or USB. It's not "open source" in the public repo sense — the source is included with your purchase, not published freely.

What's the source code license?

The source code is licensed for personal use, modification, and learning. You can customize it, build your own face styles, add integrations, and flash custom firmware. Redistribution and resale of the source code is not permitted. Think of it like buying a book — you own your copy and can do what you want with it, but you can't republish it.

What about privacy? There's a microphone in it.

Nova Sphere does not listen by default — the mic only activates when you tap the sphere. All voice processing happens on YOUR server (or locally via Ollama). No audio is sent to any cloud service unless you explicitly configure it that way. There's no account, no telemetry, no always-on listening. You control the entire pipeline.

Can I build my own face styles?

Yes. Face styles are C++ classes in the firmware. Each style defines how eyes, mouth, eyebrows, and extras are drawn. The code includes 4 built-in styles (default, OpenClaw, Claude, Ollama) that you can use as templates.
